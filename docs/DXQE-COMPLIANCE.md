## Deferred Items (with justification)

- **DX-13 automated changelog generation** (e.g. Changesets): not added. The
  project already has a clear manual `CHANGELOG.md` process documented in
  CLAUDE.md's Release Process, and introducing a changelog tool is a
  process/tooling change beyond "minimum necessary" for a project of this
  size with a single publishable package.
- **Git hooks (Husky + lint-staged)**: not added. DX-11's core requirement â€”
  automated, consistently-executable quality gates â€” is now satisfied by CI.
  Local pre-commit hooks are a DX-18 recommendation, not a DX-11
  requirement, and adding a new dependency (Husky) for a repo this size was
  judged disproportionate to the gap it closes now that CI enforces the
  same checks.
- **`install` script**: intentionally not added to package.json `scripts`.
  `install` is an npm/pnpm _lifecycle_ script name â€” defining one would hook
  into every `pnpm install` invocation rather than provide a discoverable
  command, which is a footgun, not a convenience. `pnpm install` already
  works natively as the standard entry point.

## Non-Applicable

- **DX-09 monorepo recommendation for multiple deployable artifacts**: already
  satisfied â€” Animoria is a Turborepo monorepo containing `animoria-core`,
  `animoria-vscode`, `animoria-jetbrains`, and `animoria-sandbox`.

## Known Local-Only Caveat

An untracked file `oss_bundle.md` exists at the repository root (not
version-controlled, `git status` shows it as `??`). It fails
`prettier --check` locally, which would make a naive `pnpm check` fail on
this particular machine. It does not affect CI or fresh clones, since it
isn't part of the git history. No action was taken on it since it's outside
the audited repository content and its origin/purpose is unclear.
